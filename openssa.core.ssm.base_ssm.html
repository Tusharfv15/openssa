custom_discuss(user_input: list[dict], conversation: list[dict]) → tuple[dict, list[dict]]¶

Send user input to our SLM and return the reply, AND the actual user input. In the base implementation, the user_input is unchanged from what we are given. But derived classes can override this method to do things like:

  • Add other context info to the user_input

  • Query other models first and combine their replies to form a single user_input
